## 1. 产品概述

### 1.1 产品定位

OOTD (Outfit of the Day) 是一款面向都市年轻用户的智能衣橱管理与穿搭推荐 Android 应用。通过 AI 图像识别技术帮助用户数字化管理衣橱，结合天气数据和场景需求，为用户提供个性化的穿搭建议。

### 1.2 目标用户

| 用户画像 | 特征描述 | 核心痛点 |
|---------|---------|---------|
| 都市白领 (25-35岁) | 工作繁忙，注重形象，衣橱衣物多但整理困难 | 早上时间紧，不知道穿什么 |
| 时尚爱好者 (20-30岁) | 热衷穿搭，喜欢尝试不同风格 | 衣服多但搭配灵感枯竭 |
| 学生群体 (18-25岁) | 预算有限，希望提高单品利用率 | 不会搭配，想学习穿搭技巧 |
| 极简主义者 | 追求精简衣橱，注重品质 | 需要系统管理现有衣物 |

### 1.3 核心价值

- **省时**: 3秒获取今日穿搭建议，告别"衣橱前发呆"
- **省心**: 智能管理衣橱，清晰掌握已有衣物
- **省钱**: 提高单品利用率，避免重复购买
- **专业**: AI 搭配建议，提升穿搭品味

### 1.4 产品愿景

成为用户最贴心的"数字造型师"，让每个人都能轻松穿出自信。

## 2. 用户故事

### 2.1 典型用户一：都市白领小林

**背景**: 28岁，互联网公司产品经理，工作忙碌，经常需要见客户。

**场景故事**:
> 周一早上 8:00，小林匆忙起床，今天有个重要客户会议。她打开 OOTD 应用，选择"商务场合"，应用根据今天 22°C 多云的天气，从她录入的衣橱中推荐了一套：白色衬衫 + 灰色西装外套 + 黑色西裤 + 乐福鞋。她满意地换上衣服，出门前还收藏了这套搭配以备后用。

**需求**:
- 快速场景化推荐
- 天气联动
- 收藏喜欢的搭配

### 2.2 典型用户二：大学生小美

**背景**: 21岁，大三学生，热爱时尚但预算有限。

**场景故事**:
> 周末小美要和闺蜜逛街，她打开 OOTD，把新买的连衣裙拍照录入系统，应用自动识别为"粉色碎花连衣裙，夏季，甜美风格"。她选择"休闲约会"场景，应用推荐搭配白色帆布鞋和草编包。小美发现这套搭配很清新，还学到了新的搭配技巧。

**需求**:
- 便捷的衣物录入
- 学习搭配知识
- 提高单品利用率

### 2.3 典型用户三：极简主义者阿杰

**背景**: 30岁，设计师，践行胶囊衣橱理念，衣物精简但追求质感。

**场景故事**:
> 阿杰每季度会整理一次衣橱，他用 OOTD 记录所有衣物，通过应用的统计功能发现某件外套整个冬天都没穿过，决定捐赠。应用根据他现有的 30 件单品，生成多种搭配组合，帮助他最大化利用现有衣物。

**需求**:
- 衣橱统计分析
- 单品利用率追踪
- 精简决策支持

### 3.2 模块一：衣橱管理

#### 3.2.1 拍照录入

**功能描述**: 通过相机或相册上传衣物照片，AI 自动识别衣物属性。

**用户流程**:

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 点击添加  │ -> │ 选择拍摄  │ -> │ AI识别   │ -> │ 确认保存  │
│ 衣物按钮  │    │ 或相册   │    │ 属性填充 │    │ 到衣橱   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

**详细功能**:

| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| 相机拍摄 | 调用系统相机，支持裁剪和背景虚化 | P0 |
| 相册选择 | 从相册选取，支持多选批量录入 | P1 |
| AI 识别 | 识别类别、颜色、材质、图案等 | P0 |
| 手动修正 | 用户可以修改 AI 识别结果 | P0 |
| 批量录入 | 一次选择多张照片依次处理 | P1 |

**天气-穿搭映射**:

| 温度范围 | 天气状况 | 推荐策略 |

### 5.2 图片识别技术

#### 5.2.1 技术方案

| 方案 | 优点 | 缺点 | 选型 |
|-----|------|------|------|
| 云端 AI API | 识别准确率高 | 需要联网，有隐私风险 | 备选 |
| 端侧 TFLite | 离线可用，隐私安全 | 模型大小受限 | **主选** |
| 端+云混合 | 平衡方案 | 架构复杂 | 后期优化 |

#### 5.2.2 端侧模型设计

**多任务模型架构**:

```
输入图像 (224x224x3)
    │
    ▼
┌─────────────┐
│ MobileNetV3 │  (特征提取，轻量级)
│  Backbone   │
└──────┬──────┘
       │
   ┌───┴───┬──────────┬──────────┐
   ▼       ▼          ▼          ▼
┌──────┐ ┌──────┐  ┌──────┐  ┌──────┐
│类别  │ │颜色  │  │材质  │  │风格  │
│分类头│ │识别头│  │分类头│  │标签头│
└──────┘ └──────┘  └──────┘  └──────┘
```

**模型规格**:
- 基础模型: MobileNetV3-Small (量化后 ~4MB)
- 输入尺寸: 224x224
- 推理时间: < 100ms (中端机型)
- 支持类别: 50+ 衣物类别
- 支持颜色: 20+ 基础颜色

#### 5.2.3 图像预处理

- 自动裁剪：检测衣物主体区域
- 背景虚化：减少背景干扰
- 光照归一化：提高识别稳定性

### 5.3 天气 API 集成

#### 5.3.1 服务选型

| 服务商 | 免费额度 | 特点 | 选型 |
|-------|---------|------|------|
| 和风天气 | 1000次/天 | 国内准确，文档好 | **主选** |
| OpenWeatherMap | 1000次/天 | 国际通用 | 备选 |
| 高德天气 | 5000次/天 | 定位准确 | 备选 |

#### 5.3.2 数据缓存策略

- 缓存时间: 30 分钟
- 定位精度: 城市级别
- 失败降级: 使用上次缓存数据

### 5.4 推荐算法

#### 5.4.1 算法架构

```
┌────────────────────────────────────────────────────────────┐
│                      推荐引擎                               │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐ │
│  │   过滤层     │ -> │   评分层     │ -> │   排序层     │ │
│  │  Filter      │    │  Scoring     │    │  Ranking     │ │
│  └──────────────┘    └──────────────┘    └──────────────┘ │
│         │                   │                   │         │
│  - 场景过滤            - 规则评分           - 综合排序    │
│  - 季节过滤            - 协同过滤           - 多样性控制  │
│  - 天气过滤            - 用户偏好           - 新鲜度加权  │
│  - 可用性过滤          - 历史反馈                         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 5.4.2 评分规则

**基础评分公式**:

```
Score = w1*场景匹配 + w2*天气匹配 + w3*颜色协调 + w4*用户偏好 + w5*新颖度

其中:
- 场景匹配: 衣物场合标签与目标场景的匹配度
- 天气匹配: 衣物季节/厚度与当前天气的匹配度
- 颜色协调: 基于色彩理论的配色评分
- 用户偏好: 用户对颜色/风格的偏好权重
- 新颖度: 避免重复推荐近期已穿搭配
```

**颜色协调规则**:

```kotlin
// 配色方案定义
val colorHarmonyRules = mapOf(
    "complementary" to 1.0,      // 互补色 - 高对比
    "analogous" to 0.8,          // 邻近色 - 和谐
    "monochromatic" to 0.7,      // 同色系 - 统一
    "triadic" to 0.9,            // 三色搭配
    "neutral_base" to 0.6        // 中性色基础
)
```

#### 5.4.3 冷启动处理

- 新用户：使用默认流行搭配模板
- 少量衣物：推荐单件穿搭 + 购买建议
- 持续学习：根据用户反馈调整权重
